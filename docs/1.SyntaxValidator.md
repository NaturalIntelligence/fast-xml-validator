XMLParser uses XMLValidator on demand.

```js
const {XMLParser} = require("fast-xml-parser");
const parser = new XMLParser(options);
try{
    let result = parser.parse(XMLdata, true);
}catch(err){
    //:
}
```

XML Parser throws error when XML Validator returns error. XML Validator can also be used directly without XML Parser;

```js
const {SyntaxValidator} = require("fast-xml-validator");
XMLValidator.validate(xmlData, {
    allowBooleanAttributes: true
});
```

SyntaxValidator throws validation error for unexpected syntax as per config.

## Options

### allowBooleanAttributes

Set it to `true` when a tag can have boolean attributes.

### unpairedTags
Unpaired Tags are the tags which don't have matching closing tag. Eg `<br>` in HTML. You can parse unpaired tags by providing their list to the parser, validator and builder.

```js
const xmlData = `<parent><extra></parent>`;
const result = XMLValidator.validate( xmlData, {
  unpairedTags: ["extra"]
});
```

## Comparison to Fast XML Parser

- It thorws error instead of returning the error object.
- It can validate DOCTYPE syntax too
- Since v1.1.0, element names are validated by xml-naming which also consider xml version.
- Since v1.1.0, xml declaration can also be validated.
